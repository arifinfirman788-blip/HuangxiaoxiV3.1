# 《省级旅游智能体门户APP》产品功能说明书 (PRD)

## 1. 项目概述
**项目名称**：huangxiaoxiV3.0 (黄小西 - 贵州旅游智能体门户)
**项目背景**：
本项目旨在打造一个基于多智能体（Multi-Agent）技术的省级旅游服务平台。通过“黄小西”这一核心智能体IP，聚合景区、酒店、餐饮、政府服务等多维度的垂直领域智能体，为游客提供全流程、个性化、伴随式的智慧旅游体验。解决传统OTA平台信息过载、服务被动的问题，实现“一部手机游贵州”的智能化升级。

**核心价值**：
- **主动服务**：智能体根据用户行为和对话主动推荐服务。
- **千人千面**：基于用户画像提供个性化的行程和商品推荐。
- **全域覆盖**：打通吃住行游购娱各个环节，提供一站式解决方案。

## 2. 用户角色
- **游客 (Traveler)**：主要用户，使用APP进行行程规划、咨询、预订和购物。
- **商户/运营方 (Admin/Merchant)**：管理智能体信息、商品上架、处理订单（后台功能，暂不详述）。

## 3. 功能模块详细设计

### 3.1 首页与智能体对话 (Home / Chat)
**功能描述**：APP的核心入口，采用对话式UI (CUI) 与图形化UI (GUI) 结合的方式。
- **核心智能体“黄小西”**：作为用户的贴身导游，具备统筹能力，可调度其他垂直智能体。
- **角色切换**：用户可手动切换对话对象，如“酒店助手”、“景区向导”、“美食专家”、“政务助手”。
- **快捷功能胶囊**：提供“行程规划”、“旅居管家”、“特产订购”、“私人伴游”等高频场景的快速入口。
- **语音交互**：支持语音输入，实现自然语言对话。
- **多语言支持**：支持多语言切换（如中/英），服务国际游客。

### 3.2 智能体广场 (Agent Categories)
**功能描述**：展示和索引平台上的所有垂直领域智能体。
- **分类导航**：
  - **景区智能体**：提供景点讲解、人流播报、路线规划（如：黄果树瀑布智能体、小七孔智能体）。
  - **酒店智能体**：提供房型咨询、服务预约、入住办理（如：贵阳亚朵智能体）。
  - **餐饮智能体**：提供排号、点餐、口味定制（如：老凯俚酸汤鱼智能体）。
  - **政府智能体**：提供投诉建议、旅游咨询、便民服务。
  - **旅居智能体**：提供长租公寓、生活服务信息。
- **智能体卡片**：展示智能体名称、状态（在线/忙碌）、核心技能、位置及简介。

### 3.3 智慧商城 (Shop)
**功能描述**：基于“黄小西甄选”品牌的自营及三方商品销售平台。
- **商品分类Tab**：黄小西甄选、酒店、景区、餐饮、线路。
- **特色商品**：重点销售贵州特色产品（如遵义红茶、苗族银饰、酸汤鱼套餐）。
- **商品详情**：展示商品图片、价格、评分、产地及“官方认证”标识。
- **交易闭环**：支持加入购物车、下单支付（模拟流程）。

### 3.4 智慧行程 (Trip)
**功能描述**：提供智能化的行程规划和线路推荐。
- **线路推荐**：根据季节、时长、兴趣推荐经典线路（如“黔东南民族风情游”）。
- **行程定制**：通过对话生成个性化行程表。
- **可视化地图**：在地图上展示行程轨迹和沿途兴趣点。

### 3.5 个人中心 (Profile)
**功能描述**：管理用户信息和订单。
- **用户资料**：头像、昵称、偏好设置。
- **我的订单**：查看商品、门票、酒店订单状态。
- **我的收藏**：收藏的线路、商品、智能体。

## 4. UI/UX 设计要求
- **风格**：现代、清新、具有科技感，融合贵州民族元素。
- **动效**：使用 Framer Motion 实现流畅的页面切换和组件入场动画（如卡片浮动、渐入效果）。
- **交互**：
  - 强调“对话”的即时反馈感。
  - 采用卡片式设计，信息层级分明。
  - 底部导航栏 (BottomNav) 需常驻，方便快速切换核心模块。

## 5. 技术栈
- **前端框架**：React 18
- **构建工具**：Vite
- **样式库**：Tailwind CSS
- **动画库**：Framer Motion
- **路由**：React Router v6
- **图标库**：Lucide React

## 6. 后续规划 (Roadmap)
- **Phase 1 (当前)**：完成核心UI框架、基础数据模拟、静态页面及基础交互。
- **Phase 2**：接入LLM大模型API，实现真实的智能体对话能力。
- **Phase 3**：完善后端服务，实现真实的用户系统和交易流程。
